## 当前根目录快照

- `.git/`：Git 仓库数据；不展开维护。
- `.github/`：CI 配置；`workflows/ci.yml` 运行 `tests/` 下的 unittest 套件。
- `skills/dxm/`：可独立安装的核心 skill；`VERSION` 保持 core-only doctor 版本与仓库发布版本一致，`SKILL.md` 定义四模式与完成门，`scripts/scaffold_dxm.py` 负责安全写入，`scripts/dxm_contract.py` 负责 baseline/readiness/receipt 契约，`scripts/validate_dxm.py` 提供只读 CLI，`assets/templates/` 提供五份自包含长期文档，`agents/openai.yaml` 提供当前 metadata schema。
- `skills/grilling/`、`skills/grill-with-docs/`、`skills/grill-me/`、`skills/domain-modeling/`：独立安装的 optional 增强；不属于 core DXM 运行硬依赖，其中 `grill-me` 是 legacy alias。
- `tests/test_scaffold_dxm.py`：安全写入、root/祖先路径防逃逸、baseline hydration、marker、Trellis 启动/退出语义和兼容行为。
- `tests/test_dxm_contract.py`：baseline/readiness/receipt/validator CLI 契约。
- `tests/test_policy_contract.py`：四模式、0–3 问题预算、evidence matrix、selective docs 与 frontmatter。
- `tests/test_doc_sync.py`：metadata、README、core-only packaging 以及 skill/template/script 防漂移。
- `tests/test_workflow_cases.py` / `tests/fixtures/dxm_workflow_cases.json`：绑定真实策略资产的 synthetic 行为案例、Windows/UNC 路径和隐私扫描。
- `.dxm/project.json`：本机 dogfood 的规范化项目基线；它保留绝对 root，但由 `.gitignore` 排除，不发布。共享链路文档先词法折叠路径，再只保留 `$PROJECT_ROOT` / `$ABSOLUTE_PATH` 可移植 managed block 投影。
- `.trellis/`：本机中大型任务 PRD、上下文、check、workspace 与 Trellis 工具；任务归档和 receipt 属于本地验证证据，由 `.gitignore` 排除。
- `.codex/` / `.agents/skills/`：Trellis 为 Codex 生成的本地 agents、hooks 和共享 skill 入口；不是 core DXM 或 release artifact。
- `.gitattributes` / `.gitignore`：行尾与忽略规则；模板和中文文档要求 LF，本机 `.codex/`、`.dxm/`、`.trellis/`、`.agents/` 不进入发布。
- `AGENTS.md`：本仓自身的 DXM 目录级执行规则（dogfood，2026-07-09 起）。
- `CHANGELOG.md` / `VERSION`：发布记录与当前版本号；发布流程见 `开发者AI开发与PR提交流程.md`。

## DXM 大开发工作流目录

本项目启用 Trellis/Codex 大开发工作流时，下列目录属于项目级 AI 协作基础设施：

- `.trellis/`：Trellis 项目工作流状态、任务 PRD、spec、workspace journal 和脚本。
- `.trellis/tasks/`：每个开发任务的 `task.json`、`prd.md`、实现上下文和检查上下文。
- `.trellis/spec/`：可复用项目规范；完成任务后应把稳定经验沉淀回这里。
- `.codex/`：项目级 Codex agents、hooks 和配置。
- `.agents/skills/`：跨 agent 共享的 Trellis skill 入口。

维护要求：修改这些目录的事实结构或工作流含义时，同步更新 `AGENTS.md`、`项目开发规范（AI协作）.md` 和本文档。
